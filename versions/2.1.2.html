Changes since 2.1.1-16 (2021 March 16)

General Chronojump changes

  • Top left selectors to change fast from:
    • jump simple to reactive.
    • races simple, races intervallic, race analyzer.
    • gravitatory encoder, inertial encoder.
  • On session load window you can see/hide the number or persons on that session and the number of each type of tests, by default tests or current mode are shown.
  • Now load session shows results much faster, specially when is shown again (2nd, 3rd, ... times).
  • A session can be edited and deleted from session load window.
  • Social network poll at start (just 1 time) to know what is the best channel to upload our videos. You can skip answer.
  • Chronojump uses a bit less memory because some minor tweaks where done.
  • Improvements on person select window (when persons are on top).
  • After importing a session to a new session, allows to load it in the same dialog.
  • If import file path is very long, it is displayed with dots in the middle.
  • All Chronojump exercises can be selected with left/right buttons (better for tactile screens) (previously it was only on encoder, now is in jumps, races, race analyzer, force sensor and encoder).
  • Now application title is: Chronojump - mode - session name.
  • Fixed some displaying color problems on generic windows depending on user selected background color.
  • Exporting a session can be cancelled by the user (if there is any problem or it is slow).
  • Animated logo is not shown by default on mac (until Big Sur improves).
  • Fixed highlighting always latest products icon when there is no network.
  • Improved person add/edit window.
  • Improves delete test type window: afecting al jumps and races with photocells.
  • Open a folder or file (with external program like browser of file manager) now cares on all systems about files with spaces.
  • Fixed bug on triggers on analyze graphs when there are ON and not OFF or viceversa.
  • Ensure widgets sensitivity of all the widgets is correctly applied when load a session without persons.

  • Capture feedback with animated gamified path measuring accuracy (alternative to rectangle). Watch: ChronoTalks 5 - Gamificated feedback path (worm) on Force Sensor ABD/ADD capture
  • Exercises can show repetitions: concentric, concenctric + eccentric together or separeted.
  • Fixed variability: cvRMSSD (needed sqrt and dividing by mean) (and also *100 to be percentage).
  • Added variability calculation method: RMSSD.
  • Fixed export files on certain systems (specially images on Windows and when URL has latin chars).
  • Adjust: tare/calibrate moved to center of the capture window (much clear now).
  • Fixed capture with triggers.
  • Analyze graph shows triggers also in zoom.
  • Shown triggers on analyze.
  • Import session also imports forceSensor triggers.
  • At export on creating the AB graph, it is shown vertical ablines to know AB on that set.
  • Analyze export can open result url.
  • Analyze graph fixed the X on triggers (was not exactly at place).
  • Capture horizontal/vertical lines work much better on negative values.
  • Exercise dialog has same window size for all the tabs (so clicking in next is not annoying anymore with the bottom buttons being moved all the time).
  • Do not allow to forceSensor capture if no session or no person.
  • Chronojump quits correctly if user closes software while capturing with force sensor.
  • ForceSensor capture unsensitivizes adjust button.
  • Ensure button_execute_test is not active after adjust if there are no persons.
  • RFD-back button much clear with an icon.
  • After load ensure change to signal (not rfd image).
  • Fixed CSVDecimalColumnIsPoint (fixing really eventual problems on export on R).
  • Fixed not seeing error label on forceSensorExercise create without name.

Encoder

  • Now Encoder configuration and exercises are different for gravitatory than inertial. An exercise can be gravitatory, inertial or both. Related changes:
    • button encoder config: shows gravitatory or inertia on label depending on mode.
    • Exercise edit takes care on saved sets with that exercise.
    • Displayed help button on gravitatory/inertial config problems like changing this modes on some exercise but having some sets captured on the other mode.
  • Fixed crash on inertia moment calculation since check Chronopic cable feature.
  • Analyze export can open result url.
  • Fixed wrong title "Delete exercise" on exercise edit.
  • Fixed trigger on encoder.
  • Fixed potentially bugs on select SQL methods.

Race analyzer

  • On Capture (realtime) and load, apart from a speedometer, is also shows now graphs of: speed/time, position/time, accel/time.
  • Now it is also exported: instantaneous data (raw and fitted).
  • Fixed export files on certain systems (specially images on Windows and when URL has latin chars).
  • Analyze export can open result url.
  • Speedometer shows results in two lines: Max speed and distance (also on load).
  • Now model works correctly even when the step size is too small in nls.
  • Now on load it does not change automatically to analyze tab because graphs on capture tab are relevant.
  • At load, "analyze" process is not automatically done now. User will need to click on analyze button.
  • Can capture with person height == 0 (undefined), and shows an informative message.
  • Improvements on user interface on top right of analyze tab.
  • Hidden selector for race analyzer (resisted) until product is finished.

Jumps, Races

  • Last Jump simple or multiple configuration is remembered when Chronojump is closed. Work faster and avoid mistake. Example of variables remembered: Sjl weight, Dj fall number of jumps, number of seconds, ...
    • Sprints can be exported like forceSensor and RaceAnalyzer
    • Export all the sprints of a person or all the sprints on the session.
    • Includes images and table (summarized spreadsheet data).
    • Export includes laps, and laps can be different on different sets (needed columns will be automatically created on CSV).
    • Export files managed correctly on all systems (specially images on Windows and when URL has latin chars).
  • Fixed ending of RSA tests since last versions, now end time is ok.
  • Datetime is shown in races,a also on export.
  • On jumps multiple results table, it is shown now also the average of power and stiffness..
  • Simple races new analyze graph: Races/time. This graph can be saved, it can show all or only best in session, can show speed or time, km/h or m/s, has a selector for tests with custom (selectable) distances (and remember these in preferences).
  • Jumps/Races simple capture tab graph can be saved.
  • Intervallic race types with different distances allow 5 characters: eg 25,51. So now decimal values are available on this tipe or races.
  • Do not allow to press results table buttons while jump/race capture.
  • Fixed deleting last contact time on ending of an Rj jump (fixes crash on executing unlimited jumps with just 1 jump).
  • Automatic jumps sequence does not allow to add exercises on load (because it was confusing, but continue thinking on better gui and all tests integration).
  • Better animation on moving bar of last simple jump / simple race done. Animation according elapsed time.
  • Jump fall entry button has now 1 decimal.
  • Shown decimal separator character on adding a new race type.
  • Fixed not showing track distance on creating new simple race type.
  • Intervalic races show total in bold.
