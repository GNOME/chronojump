General Chronojump changes

  • Tags: Sessions can have user created tags, allowing users to organize much better their data. Tags can be edited/deleted, sessions can be filtered/sorted by tags. Tags cannot be imported/exported from another database.
  • Improved import session, now using Python 3 for all systems. This fixes occasional problems with non-English characters.
  • Improvements on overview window: force sensor, encoder, race analyzer. Current person is selected on the data table and selecting the desired person is easier.
  • New button: Latest products, will show new Chronojump products on a non-intrusive way.
  • Scroll to row on persons list at left when selecting a person from overview window.
  • Improvements on exporting a session.
  • Persons top window improvements: can select and close a window double clicking on person, displays correctly long names, bigger up/down buttons that displaces list row by row, and has better behaviour on selecting person.
  • Removed request to confirm on exit.
  • Rest time (minutes/seconds) is saved to be used automatically on next sessions.
  • Improved send error log window.
  • Better problem solving if user disconnected Chronopic.
  • New animated logo for Chronojump 2.1.x (versions of 2021).
  • Fixed session comments editing.

Jumps, Races

  • RSI on jumps reactive (jump height / contact time).
  • When opening Chronojump, the first click on execute jumps or races will connect with the Chronopic.
    On past versions it connected and started the jump and it was a bit confusing on some situations.
  • Better start detection on sprint model detecting initial force by time-shifting method.
  • On races with 2 tracks do not allow to do automatic sprint calculations (because model is not enough accurate for just two tracks).
  • Jump profile graph improvements: When some of the indexes are not calculated a white area is shown on the graph. Image can be saved even if not all the indexes are calculated. Better display of problems like a low Dj height.
  • Improved intervallic races graph.
  • On jump simple creation renamed types: normal/drop to jump/rebound.
  • Clarified options on creation of a jump reactive test.

Force sensor

  • New features on analyze.
    • On series of different repetitions, user can choose which repetition to analyze with the model.
    • Graphical interface: Changes on how raw and modelized analysis are shown. Bigger A,B sliders.
    • Can start/end analysis on a fixed millisecond or let Chronojump find the best start/end on some A,B range.
  • Improvements on modelized data calculations and start/end detection.
  • Implemented CVRMSSD on force sensor variability (this method can be selected on preferences).
  • Analyze raw data graph displays now: force, rfd max of selected AB. nice for having relevant data at export image.
  • On analyze can move the A, B sliders (1s distance), and this value can be configured on preferences.
  • Show laterality icon at top.
  • Fixed: export/import of forceSensor files from systems with different decimal separator.
  • Fixed laterality reading on load set.
  • Fixed showing "Free" test after changing from jump mode to force sensor mode.
  • Fixed few potential problems on trying to analyze a set incorrectly captured.

Encoder

  • New calculation for inertial: Equivalent mass. Uses inertia moment, diameter and force multiplier, can be crossed with other variables on analyze.
  • On inertial show equivalent mass and diameter on analyze data table.
  • Encoder work and impulse shown real time (on previous version was shown at the end of the capture).
  • Capture graph (bars) can be saved
  • On analyze superpose, shown laterality if is different o selected repetitions.
  • On capture showing loss can be disabled.
  • Improved decimal places of some variables on export.
  • On gravitatory now do not show inertia moment on data table.
  • Better display of preferences / encoder.
  • Better management of buttons when going to capture/analyze and backwards.
  • Fixed: Encoder, instantaneous analysis graphic size is wrong at first time.

Race analyzer

  • At capture on realtime it shows a velocimeter graph, and variables: speed and distance.
  • Shown an exportable data table for race analyzer results.
  • User can adjust the PPS (Pulses per Sample) on capture, allowing to change space resolution on capture. Before space resolution was 3 cm, now it can be configured from 3 mm to 3 m.
  • Improved starting point detection.

Chronojump-networks

  • Added button for not upload a capture (eg. for warming up).
  • Networks person auto logout will close the encoder exercise config.
  • Fixed system to 2021 server code.

Translations

  • Updated translations: Brazilian Portuguese, Catalan, Czech, Romanian, Spanish, Swedish, Ukrainian.
